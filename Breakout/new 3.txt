Idea: Use a tablet, or phone, to control Python​.

In this scenario Python becomes both the client and server.

Test case: A Breakout​ game using the pyGame​ library.

The game used was a group project at UMGC​.
The group consisted of: Christopher Baquet​, Andre Dothlyn​, Tori Haynes​, Rick Lyman​ and Derrick Perry​ .

This particular implementation will use web sockets via a web browser. 
It's germination was the investigation of running the game using WASM​.

Running Python using WASM is doable​; however, pyGame complicates things.
Using Pygbag​ is a solution for doing this. To run across the internet, this approach will eventually be investigated.

For now, this author is interested in running on a local machine; so, working with Pygbag is deferred.
Currently the local machine is running a full OS (Mac, Windows, or Linux).

The author would like to move from an OS to bare metal​. However, that day, is not today.

The tablet, or phone, will use a web browser to record touch events​.
